# Pitch Checklist

## Slide Order

1. Problem
2. Why existing solutions fail
3. Why blockchain is necessary here
4. Product flow
5. Demo
6. Impact and deployment path
7. Responsible design
8. Ask / closing

## Each Slide Must Prove Something

- Problem:
  誰がどの現場で困っているか
- Existing gap:
  なぜ未解決か
- Blockchain utility:
  何を trust layer として担うか
- Demo:
  何が今すでに動くか
- Deployment path:
  誰と試せるか
- Impact:
  何がどれだけ改善するか

## Demo Checklist

- オフラインまたは不安定な回線でも見せられる
- ブロックチェーンを使った箇所が画面で確認できる
- 正常系が 90 秒以内で通る
- 失敗系を 1 つ以上見せられる
- ウォレット接続なしで成立する
- 個人情報をオンチェーンに保存していないと説明できる

## Judge Questions To Pre-answer

- なぜ普通の DB ではだめか
- 誰が導入主体か
- 既にある代替手段との差は何か
- 実証するとしたら最初のパートナーは誰か
- 不正入力や誤判定にどう対処するか

## Final Submission Pack

- README
- 2 分以内のデモ動画
- スライド
- リポジトリ
- アーキテクチャ図
- 審査員向け 1 段落要約
