# Idea Scorecard

## Scoring Rule

各項目を 1-5 点で採点する。合計 40 点満点。

- Problem severity
- User clarity
- Evidence strength
- Blockchain necessity
- Practicality in a hackathon
- Demo clarity
- BGA fit
- Distinctiveness

## Interpretation

- 34-40: 本命候補
- 28-33: 伸びる余地あり
- 27 以下: 捨て候補

## Kill Questions

以下のどれかに `Yes` なら、原則として却下する。

- その課題は「便利そう」止まりで、痛みが弱いか
- blockchain を外しても価値がほぼ同じか
- 審査員が 30 秒で理解できないか
- 実ユーザーが crypto UX を要求されるか
- 既存のハッカソン案と見分けがつきにくいか
