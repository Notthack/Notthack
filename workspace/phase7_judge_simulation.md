# Phase 7 — Judge Simulation

## Judge Lens

Score the project the way a skeptical BGA judge would:

- real-world problem clarity
- blockchain necessity
- practicality
- impact potential
- UX for non-crypto users
- responsible design
- deployment path
- demo reliability
- scope discipline

## Hard Scorecard

| Dimension | Score / 5 | Judge reading |
| --- | --- | --- |
| Real-world problem clarity | 5 | Strong. The pain is concrete and easy to explain. |
| Blockchain necessity | 4 | Strong, but only if the shared-state story is explicit. |
| Practicality | 5 | Strong. A university pilot is believable. |
| Impact potential | 4 | Good. Smaller than national aid, but very legible. |
| Non-crypto UX | 5 | Strong. Wallet-free QR flow is correct. |
| Responsible design | 5 | Strong. PII stays off-chain. |
| Demo reliability | 4 | Good. Acceptance passes, but demo choreography still matters. |
| Deployment path | 4 | Good. Campus pilot is plausible. |
| Distinctiveness | 3 | Weakest dimension. This can still look like a voucher admin tool unless framed carefully. |

## What Still Looks Weak

1. The product can still be mistaken for a generic voucher management app if the pitch does not emphasize shared trust state.
2. The blockchain moment is present but easy to miss if the auditor/history step is not highlighted.
3. The local ledger abstraction is acceptable for demo safety, but it is not inherently proof of blockchain use unless the narration is disciplined.
4. The interface is functional, but not visually iconic. It is operational, not unforgettable.
5. The scope is strong, but the story is not yet automatically obvious to a judge in the first 10 seconds.

## Why It Is Still Competitive

- The problem is real and institutionally grounded.
- The beneficiary UX is non-crypto and low friction.
- The trust model is narrow and defensible.
- The demo has a clear failure case that judges understand instantly.
- The deployment path is credible, not speculative.

## Why It Could Lose

- If the team talks like it built “a better voucher system” instead of “a shared trust layer.”
- If the audit view is treated as decorative.
- If the blockchain explanation becomes a tutorial instead of a necessity argument.
- If the live demo stumbles on state reset or duplicate redemption.

## Judge Verdict

This is finalist-credible if the team keeps the narrative tight and the live demo clean.  
It is not winning by novelty; it wins by being legible, practical, and hard to dismiss.
