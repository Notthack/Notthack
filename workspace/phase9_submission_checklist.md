# Phase 9 — Submission Checklist

## Core Assets

- demo video or live demo recording
- slide deck
- repository with runnable local demo
- README with run instructions
- architecture overview
- short written explanation of blockchain necessity
- deployment path summary
- privacy / responsible design note

## Demo Requirements

- merchant verify works
- merchant redeem works
- duplicate redemption is blocked
- revoked voucher is blocked
- unauthorized merchant is blocked
- student flow remains wallet-free
- auditor can inspect meaningful history

## Narrative Requirements

- one clear problem
- one clear user group
- one clear reason current workflow fails
- one clear blockchain necessity argument
- one clear pilot path
- one clear responsible-design story

## Final Review Checks

- the demo story is understandable in under 2 minutes
- the UI does not feel like a generic admin panel
- the blockchain moment is visible but not overexplained
- the scope stays narrow and judge-friendly
- the wording avoids overclaiming about production blockchain

## Submission Order

1. rehearse demo
2. capture final screenshots or video
3. verify no contradictions between slide, demo, and README
4. package links and short summary
5. submit
